# Inbox Management Spec

## 概述

Inbox Management 是 Context Inbox 功能的核心管理模块，负责未处理上下文的存储、查询、更新和删除操作，并提供分页和过滤功能。

## 功能描述

### 1. Inbox 存储

#### 1.1 创建 Inbox 项
- **描述**：将用户捕获的上下文添加到 Inbox 中
- **API**：`add_to_inbox(context, source)`
- **请求参数**：
  - `context`：字符串，要存储的上下文文本
  - `source`：Source 类型，上下文的来源信息（包含类型、标题、URL 等）
- **返回值**：创建的 InboxItem 对象

#### 1.2 查询 Inbox 项
- **描述**：获取 Inbox 中的所有项，支持分页和过滤
- **API**：`get_inbox_items(page, page_size, search, source_type, processed)`
- **请求参数**：
  - `page`：整数，页码（从 0 开始）
  - `page_size`：整数，每页大小（默认 10）
  - `search`：字符串，可选，搜索关键词
  - `source_type`：字符串，可选，源类型（web/video/pdf/app）
  - `processed`：布尔值，可选，是否显示已处理的项
- **返回值**：InboxItem 对象数组

#### 1.3 删除 Inbox 项
- **描述**：从 Inbox 中删除一个项
- **API**：`delete_inbox_item(item_id)`
- **请求参数**：
  - `item_id`：字符串，要删除的项的 ID
- **返回值**：布尔值，表示操作是否成功

#### 1.4 标记为已处理
- **描述**：将一个项标记为已处理
- **API**：`mark_inbox_item_processed(item_id, notes)`
- **请求参数**：
  - `item_id`：字符串，项的 ID
  - `notes`：字符串，可选，处理说明
- **返回值**：布尔值，表示操作是否成功

#### 1.5 清空已处理的项
- **描述**：删除所有已处理的项
- **API**：`clear_processed_items()`
- **返回值**：布尔值，表示操作是否成功

### 2. 上下文处理

#### 2.1 处理 Inbox 项
- **描述**：处理一个 Inbox 项，为其关联一个或多个单词
- **API**：`process_inbox_item(item_id, lemmas)`
- **请求参数**：
  - `item_id`：字符串，要处理的项的 ID
  - `lemmas`：字符串数组，要关联的单词原型列表
- **返回值**：创建或更新的 Note 对象数组

**实现约束（本次版本）**：
- 允许重复 Context：当一个 Inbox 项被多个 lemma 处理时，会为每个 lemma 创建一条新的 `contexts` 记录（sentence/source 可重复）
- 不要求“共享同一个 Context”跨 Note 复用

#### 2.2 单词提取
- **描述**：从上下文中自动提取可能的单词
- **API**：`extract_words(context)`
- **请求参数**：
  - `context`：字符串，要提取单词的上下文
- **返回值**：提取的单词字符串数组

#### 2.3 单词提取（带频率）
- **描述**：从上下文中自动提取可能的单词，并返回其出现频率
- **API**：`extract_words_with_frequency(context)`
- **请求参数**：
  - `context`：字符串，要提取单词的上下文
- **返回值**：包含单词和频率的元组数组

### 3. 搜索和过滤

#### 3.1 搜索功能
- **描述**：根据关键词搜索 Inbox 中的项
- **搜索范围**：
  - 上下文文本
  - 源标题（如网页标题、文件名）
- **搜索方式**：
  - 模糊搜索
  - 支持大小写不敏感搜索
  - 按匹配得分排序

#### 3.2 过滤功能
- **描述**：根据源类型和处理状态过滤 Inbox 项
- **源类型过滤**：
  - web（网页）
  - video（视频）
  - pdf（PDF 文档）
  - app（其他应用程序）
- **处理状态过滤**：
  - 所有项
  - 未处理的项
  - 已处理的项

### 4. 分页功能

#### 4.1 分页配置
- **默认配置**：每页显示 10 个项
- **支持的分页大小**：5、10、20、50、100

#### 4.2 分页信息
- **API 响应**：
  - `items`：当前页的项数组
  - `current_page`：当前页码（从 0 开始）
  - `total_pages`：总页数
  - `total_items`：总项数

### 5. 数据导入/导出

#### 5.1 导出功能
- **描述**：将 Inbox 内容导出为 CSV 或 JSON 格式
- **导出格式**：
  - CSV：包含所有字段的表格
  - JSON：结构化的数组格式

#### 5.2 导入功能
- **描述**：从外部文件导入 Inbox 内容
- **支持的格式**：
  - CSV（逗号分隔值）
  - JSON（JavaScript 对象表示法）

## 边界条件

### 1. 空上下文
- **情况**：用户提交空字符串或仅包含空格的上下文
- **处理**：
  - 拒绝存储
  - 返回错误信息："Context cannot be empty"

### 2. 无单词提取
- **情况**：上下文中没有可提取的单词
- **处理**：
  - 允许存储到 Inbox
  - 处理时提示用户手动输入单词

### 3. 处理失败
- **情况**：处理 Inbox 项时发生错误（如数据库连接失败）
- **处理**：
  - 回滚操作
  - 保持项的未处理状态
  - 显示详细错误信息

### 4. 重复处理
- **情况**：同一 Inbox 项被处理多次
- **处理**：
  - 检查项是否已处理
  - 已处理的项不允许再次处理
  - 显示已处理的提示

## 性能考虑

### 1. 查询优化
- **索引策略**：
  - 在 `captured_at` 字段上添加索引以优化按时间排序
  - 在 `context` 字段上添加全文搜索索引
  - 在 `processed` 和 `source_type` 字段上添加复合索引

### 2. 内存使用
- **分页限制**：避免一次加载大量项到内存
- **流式处理**：对于大型 Inbox，使用流式查询

### 3. 异步操作
- **处理异步化**：长时间运行的操作（如大量项的处理）使用异步方式
- **进度反馈**：提供处理进度的实时反馈

## 安全考虑

### 1. 输入验证
- **SQL 注入防护**：对所有用户输入进行参数化查询
- **XSS 防护**：存储和显示时对 HTML 内容进行转义
- **文件导入验证**：检查导入文件的格式和内容安全

### 2. 权限控制
- **用户隔离**：确保每个用户只能访问自己的 Inbox
- **访问控制**：防止未经授权的 API 调用

### 3. 数据备份
- **定期备份**：建议用户定期备份数据库
- **恢复机制**：提供数据恢复功能

## 版本兼容性

### 1. 向后兼容性
- **与现有功能的兼容性**：Inbox 功能与现有的捕获和管理功能完全兼容
- **数据迁移**：对于升级的用户，提供平滑的数据迁移路径

### 2. 版本控制
- **API 版本控制**：对所有外部 API 进行版本控制
- **变更通知**：当 API 发生破坏性变更时，提供足够的通知和文档
