# Inbox UI Spec

## 概述

Inbox UI 提供了一个直观的界面，用于管理用户捕获到 Context Inbox 中的上下文，并支持为每个上下文选择单词和处理操作。

## 页面结构

### 1. 导航布局

#### 1.1 顶部导航
- **标题**："Context Inbox"
- **统计信息**：显示未处理和已处理的上下文数量
- **操作按钮**：
  - [快速捕获]：打开快速捕获弹窗
  - [清空已处理]：删除所有已处理的上下文
  - [导出]：将 Inbox 内容导出为 CSV/JSON

#### 1.2 过滤和搜索栏
- **搜索框**：
  - 实时搜索功能
  - 支持高亮匹配
  - 搜索范围：上下文内容和源信息
- **源类型过滤器**：
  - 下拉选择源类型
  - 支持多选
- **处理状态过滤器**：
  - 所有
  - 未处理
  - 已处理

### 2. 主要内容区域

#### 2.1 上下文列表视图
- **单个 Inbox 项**：
  - **上下文预览**：显示前几行，超出部分截断
  - **源信息**：显示源类型图标和标题
  - **捕获时间**：显示捕获的相对时间
  - **操作按钮**：
    - [处理]：打开处理对话框
    - [删除]：删除该上下文
- **项状态指示**：
  - 未处理：正常显示
  - 处理中：显示加载指示器
  - 已处理：显示已处理标记

#### 2.2 空状态
- **无上下文时**：
  - 显示友好的提示信息
  - 说明如何添加上下文
  - 提供快速捕获入口

#### 2.3 加载状态
- **初始加载**：显示骨架屏加载
- **分页加载**：显示加载指示器
- **错误状态**：显示错误信息和重试按钮

### 3. 分页控制

#### 3.1 页码导航
- **页码显示**：显示当前页码和总页数
- **跳转控件**：
  - 上一页/下一页按钮
  - 跳转到第一页/最后一页
  - 页码输入框

#### 3.2 每页显示数量
- **下拉选择**：允许用户选择每页显示的数量
- **选项**：5、10、20、50、100

### 4. 上下文处理对话框

#### 4.1 对话框布局
```
┌─────────────────────────────────────────────────────┐
│ Process Context                                     │
├─────────────────────────────────────────────────────┤
│ Context Preview:                                    │
│ "The implementation details of the WittCore ..."    │
│                                                     │
│ Words to Capture:                                  │
│ [   ] implementation                                │
│ [   ] details                                       │
│ [   ] WittCore                                      │
│ [ + ] Add custom word...                            │
│                                                     │
│ [Extract from Context] [Select All] [Deselect All]  │
│                                                     │
│ [Process] [Cancel]                                  │
└─────────────────────────────────────────────────────┘
```

#### 4.2 单词选择区域
- **自动提取的单词列表**：
  - 复选框允许用户选择单词
  - 显示每个单词在上下文中的位置
  - 支持点击高亮显示

#### 4.3 手动添加单词
- **添加按钮**：显示输入框
- **手动输入**：允许用户输入未被提取的单词
- **验证**：确保输入的是有效的单词格式

#### 4.4 单词提取控制
- **重新提取**：允许用户重新提取单词
- **全部选择/取消**：提供批量操作
- **单词计数**：显示已选择的单词数量

#### 4.5 处理操作
- **处理按钮**：点击后开始处理
- **处理进度**：显示处理的进度和状态
- **错误处理**：显示处理过程中可能出现的错误

## 交互设计

### 1. 响应式设计

#### 1.1 断点设计
- **桌面端**：完整布局
- **平板端**：调整布局适应较小屏幕
- **移动端**：简化布局，优化触摸操作

#### 1.2 触摸优化
- **增加点击目标**：所有交互元素至少 48x48px
- **间距调整**：增加元素间的间距
- **手势支持**：支持常见的触摸手势

### 2. 动画和过渡

#### 2.1 页面动画
- **加载动画**：骨架屏加载动画
- **内容出现**：渐进式动画
- **过滤动画**：平滑的高度变化

#### 2.2 交互反馈
- **悬停状态**：显示元素的交互状态
- **点击反馈**：点击时的视觉反馈
- **成功/失败反馈**：操作后的反馈动画

### 3. 键盘导航

#### 3.1 基础导航
- **Tab 键**：在交互元素间导航
- **Shift+Tab**：反向导航
- **Enter/Return**：激活当前元素

#### 3.2 快捷键支持
- **/ 或 Ctrl+K**：聚焦搜索框
- **Ctrl+Shift+A**：全选
- **Ctrl+Shift+D**：取消全选
- **Ctrl+Delete**：删除选中项

### 4. 可访问性设计

#### 4.1 语义化 HTML
- **使用适当的标签**：h1-h6, button, input 等
- **ARIA 属性**：为非标准组件添加 ARIA 属性
- **语义化图标**：为图标添加适当的描述

#### 4.2 对比度
- **文本对比度**：至少 4.5:1（WCAG 2.1 AA 级）
- **交互元素对比度**：至少 3:1

#### 4.3 文字大小
- **可缩放性**：允许用户调整文字大小
- **响应式文字**：根据设备尺寸调整文字大小

## 视觉设计

### 1. 配色方案

#### 1.1 基础颜色
- **背景色**：#FFFFFF（白色）
- **前景色**：#1F2937（深灰色）
- **次要文本**：#6B7280（灰色）

#### 1.2 状态颜色
- **未处理**：#0EA5E9（蓝色）
- **处理中**：#F59E0B（橙色）
- **已处理**：#10B981（绿色）
- **错误**：#EF4444（红色）

### 2. 字体和排版

#### 2.1 字体层级
```
Context Title: 2xl, bold, text-gray-900
Context Preview: base, text-gray-700
Source Info: sm, text-gray-500
```

#### 2.2 间距
- **元素间距**：使用 0.5rem、1rem、2rem 等单位
- **内容区域**：确保有足够的留白
- **响应式间距**：在不同屏幕尺寸上调整

### 3. 图标设计

#### 3.1 源类型图标
- **网页**：🌐
- **视频**：🎬
- **PDF**：📄
- **应用**：💻

#### 3.2 操作图标
- **处理**：⚙️ 或 🛠️
- **删除**：🗑️ 或 🚮
- **添加**：➕ 或 ➕

## 技术实现

### 1. 组件架构

```typescript
// 主要组件
- InboxPage          // 主页面组件
  - InboxHeader      // 页面头部
  - SearchBar        // 搜索栏
  - FilterBar        // 过滤器
  - ContextList      // 上下文列表
  - ContextItem      // 单个上下文项
  - EmptyState       // 空状态
  - Pagination       // 分页组件
  - ProcessingDialog // 处理对话框
```

### 2. 状态管理

```typescript
interface InboxUIState {
  items: InboxItem[];
  isLoading: boolean;
  searchTerm: string;
  sourceTypeFilter: string[];
  processedFilter: 'all' | 'processed' | 'unprocessed';
  currentPage: number;
  pageSize: number;
  selectedItems: string[];
  isProcessing: boolean;
  processingItem: string | null;
}
```

### 3. 数据流

```
// 数据获取流程
InboxPage → useInboxStore.loadItems() → Tauri Command → witt-core → SQLite
                                                                   ↓
InboxPage ← useInboxStore.items ← Tauri Event ← witt-core ← SQLite
```

## 性能优化

### 1. 渲染优化

#### 1.1 虚拟化列表
- **实现**：使用 react-window 或 react-virtual
- **触发条件**：当项数超过 50 时启用

#### 1.2 懒加载
- **图片懒加载**：只在需要时加载图片
- **组件懒加载**：动态导入非关键组件

### 2. 网络优化

#### 2.1 分页加载
- **实现**：使用无限滚动或传统分页
- **策略**：预加载下一页数据

#### 2.2 缓存策略
- **数据缓存**：对 API 响应进行本地缓存
- **失效策略**：在数据变更时更新缓存

### 3. 代码优化

#### 3.1 组件优化
- **使用 memo**：防止不必要的重新渲染
- **拆分组件**：将大型组件拆分为更小的组件
- **避免内联函数**：避免在渲染过程中创建新函数

## 边界条件

### 1. 大量数据

#### 1.1 性能处理
- **优化查询**：确保数据库查询高效
- **虚拟列表**：避免大量 DOM 元素
- **分页加载**：防止内存问题

#### 1.2 用户体验
- **进度反馈**：提供足够的加载反馈
- **数据概览**：显示统计信息帮助用户理解数据量

### 2. 处理操作失败

#### 2.1 错误恢复
- **重试机制**：允许用户重试失败的操作
- **状态管理**：确保 UI 状态与实际数据一致

#### 2.2 错误信息
- **清晰的错误信息**：解释失败的原因
- **建议的解决方法**：提供下一步操作建议

### 3. 并发操作

#### 3.1 数据一致性
- **乐观更新**：在客户端显示预期状态
- **冲突检测**：检测并解决数据冲突
- **回滚机制**：在操作失败时回滚状态

## 与其他功能的集成

### 1. 与快速捕获的集成

#### 1.1 入口点
- **页面按钮**：提供快速捕获入口
- **对话框**：与 Quick Capture 组件共享逻辑

#### 1.2 数据同步
- **实时更新**：捕获后立即更新 Inbox 列表
- **通知**：显示捕获成功的通知

### 2. 与库视图的集成

#### 2.1 导航
- **页面切换**：提供从 Inbox 到库视图的导航
- **共享状态**：确保状态一致性

#### 2.2 功能扩展
- **批量操作**：支持选择多个上下文进行操作
- **导出功能**：支持将处理后的上下文导出到库

### 3. 与搜索功能的集成

#### 3.1 搜索集成
- **统一搜索**：与库搜索功能共享代码
- **上下文搜索**：支持在 Inbox 上下文中搜索

#### 3.2 搜索结果
- **高亮显示**：在 Inbox 项中高亮显示匹配的文本
- **快速操作**：提供对搜索结果的快速操作

## 安全考虑

### 1. 输入验证

#### 1.1 搜索输入
- **SQL 注入防护**：对搜索输入进行适当的处理
- **XSS 防护**：对搜索结果进行转义处理

#### 1.2 表单输入
- **验证规则**：对所有用户输入进行验证
- **防垃圾邮件**：限制提交频率

### 2. 权限控制

#### 2.1 访问控制
- **用户隔离**：确保用户只能访问自己的 Inbox
- **权限验证**：检查用户权限

#### 2.2 数据安全
- **加密传输**：确保数据在传输过程中的安全
- **存储加密**：对敏感数据进行加密存储
