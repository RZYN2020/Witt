# Quick Capture Spec

## 概述

Quick Capture 是 Context Inbox 功能的快速入口，允许用户使用单一快捷键快速捕获上下文，而无需立即选择单词。该功能设计为最小化对阅读体验的干扰。

## 功能描述

### 1. 启动方式

#### 1.1 全局快捷键
- **默认快捷键**：`Ctrl+Alt+I`（可配置）
- **作用**：立即打开快速捕获弹窗
- **平台兼容性**：
  - Windows：`Ctrl+Alt+I`
  - macOS：`Cmd+Option+I`
  - Linux：`Ctrl+Alt+I`

#### 1.2 其他启动方式
- **应用程序内启动**：通过菜单或按钮启动
- **命令行启动**：支持通过命令行参数启动

### 2. 捕获流程

#### 2.1 快速捕获弹窗
- **设计理念**：极简设计，减少认知负担
- **主要元素**：
  - 上下文输入区域（自动聚焦）
  - 源信息显示（自动提取）
  - 操作按钮（保存到 Inbox / 取消）
- **界面布局**：
  ```
  ┌─────────────────────────────────────────────────────┐
  │ Context Inbox - Quick Capture                        │
  ├─────────────────────────────────────────────────────┤
  │                                                     │
  │ Context: [Some text with interesting words... ]     │
  │                                                     │
  │ Source: [Web: Example Article]                      │
  │                                                     │
  │ [Save to Inbox] [Cancel]                            │
  └─────────────────────────────────────────────────────┘
  ```

#### 2.2 自动源信息提取
- **支持的源类型**：
  - **浏览器**（Chrome, Firefox, Safari）：
    - 网页标题
    - 网址
    - 网页图标
  - **视频播放器**：
    - 视频文件名
    - 时间戳
    - 帧信息（可选）
  - **PDF 阅读器**：
    - PDF 文件名
    - 页码
  - **其他应用程序**：
    - 应用程序名称
    - 窗口标题（可选）

#### 2.3 自动聚焦
- **弹窗打开后**：上下文输入区域自动获得焦点
- **键盘导航**：
  - `Tab`：在输入框和按钮之间导航
  - `Enter`：保存到 Inbox
  - `Escape`：取消操作

### 3. 源信息提取

#### 3.1 浏览器集成
- **Chrome 扩展**：通过 Chrome 扩展获取页面信息
- **浏览器标题栏**：使用原生 API 获取活动标签页信息
- **网址获取**：通过剪贴板或其他方式获取

#### 3.2 视频播放器集成
- **系统级集成**：通过操作系统 API 获取当前活动窗口信息
- **媒体播放器支持**：支持常见的媒体播放器（VLC, MPC-HC, QuickTime 等）
- **时间戳提取**：通过播放器 API 获取当前播放位置

#### 3.3 截图功能
- **可选功能**：允许用户在捕获时添加截图
- **自动截图**：在支持的源中自动捕获帧图像
- **图像优化**：压缩图像以减少存储需求

### 4. 内容优化

#### 4.1 文本格式化
- **自动格式化**：对粘贴的文本进行基本格式化
- **换行处理**：自动调整文本换行
- **空白压缩**：压缩多余的空白字符

#### 4.2 字符编码
- **自动检测编码**：处理各种字符编码
- **Unicode 支持**：完整支持 Unicode 字符
- **语言检测**：自动识别文本语言（用于后续单词提取）

### 5. 错误处理

#### 5.1 上下文处理错误
- **无效文本**：文本包含无效字符
- **网络错误**：无法获取源信息（如网页元数据）
- **权限错误**：应用程序无权限访问源信息

#### 5.2 存储错误
- **数据库连接失败**：无法连接到 SQLite 数据库
- **磁盘空间不足**：存储设备剩余空间不足
- **文件权限错误**：应用程序无写入权限

### 6. 性能优化

#### 6.1 快速响应
- **弹窗显示时间**：从快捷键到界面显示 < 500ms
- **保存操作**：保存到 Inbox 的操作 < 200ms
- **源信息提取**：后台异步提取源信息，不阻塞界面

#### 6.2 资源消耗
- **内存使用**：保持在合理范围内
- **CPU 消耗**：避免不必要的计算
- **网络请求**：最小化网络请求（如网页信息提取）

### 7. 可访问性

#### 7.1 键盘导航
- **完整支持**：所有功能可通过键盘访问
- **快捷键提示**：界面元素显示可用的快捷键
- **焦点管理**：清晰的焦点指示

#### 7.2 屏幕阅读器支持
- **ARIA 标签**：所有交互元素都有适当的 ARIA 标签
- **语义化 HTML**：使用语义化标签提高可访问性
- **对比度**：符合 WCAG 2.1 AA 级对比度要求

### 8. 配置选项

#### 8.1 快捷键配置
- **可配置性**：允许用户自定义快捷键
- **冲突检测**：检测与其他应用程序的快捷键冲突
- **恢复默认**：提供恢复默认快捷键的选项

#### 8.2 行为配置
- **自动源信息提取**：允许用户启用/禁用源信息自动提取
- **截图功能**：允许用户启用/禁用自动截图
- **文本格式化**：允许用户配置文本格式化选项

## 边界条件

### 1. 空上下文
- **情况**：用户未输入任何文本或仅包含空格
- **处理**：
  - 保存按钮禁用
  - 显示提示："Context cannot be empty"

### 2. 长上下文
- **情况**：用户输入的上下文非常长
- **处理**：
  - 限制最大字符数（如 5000 字符）
  - 显示字符计数器
  - 提供文本截断提示

### 3. 源信息获取失败
- **情况**：无法从当前源获取信息
- **处理**：
  - 使用默认源类型（app）
  - 允许用户手动输入源信息

### 4. 浏览器集成问题
- **情况**：无法与浏览器建立通信
- **处理**：
  - 使用剪贴板中的内容
  - 允许用户手动输入源信息

## 与其他功能的集成

### 1. 与主捕获功能的关系
- **互补功能**：
  - Quick Capture（Ctrl+Alt+I）：快速捕获到 Inbox
  - Normal Capture（Ctrl+Alt+C）：立即捕获并绑定单词
- **转换方式**：
  - Inbox 中的项可以转换为 Note
  - 支持将 CaptureRequest 转换为 InboxItem

### 2. 与库视图的集成
- **入口点**：库视图中的 Inbox 标签页
- **状态同步**：Inbox 内容在库视图中实时更新
- **批量操作**：支持从库视图进行批量处理

## 安全考虑

### 1. 输入验证
- **SQL 注入防护**：对所有输入进行参数化处理
- **XSS 防护**：对显示的内容进行适当的转义
- **恶意内容检测**：识别并处理恶意文本内容

### 2. 权限管理
- **最小权限原则**：只请求必要的系统权限
- **权限说明**：清晰说明请求权限的原因
- **权限降级**：当权限被拒绝时，提供替代功能

### 3. 数据加密
- **存储加密**：对存储的数据进行加密
- **传输加密**：对网络通信进行加密（如 HTTPS）
- **密码保护**：支持应用程序密码保护
